## Release Notes

Rule-code anchor links in review comments now navigate directly to the correct rule section instead of dropping readers at the top of the rules document.

## 🐛 Bug Fixes

### Broken Rule-Code Anchor Links in Review Output

When the AI code review posted comments referencing specific rules (e.g. `check-naming`, `check-complexity`), the links were generating uppercase fragment anchors that GitHub couldn't match to any heading — so clicking them landed at the top of the rules document rather than the relevant rule section. Anchor fragments are now always lowercased, matching GitHub's heading anchor format, so rule-code links in both review comments and replies resolve to the correct section.
