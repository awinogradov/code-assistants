## [1.9.0](https://github.com/awinogradov/code-assistants/compare/autopilot@v1.8.1...autopilot@v1.9.0) (2026-07-10)

## Release Notes

The plan skill now enters plan mode automatically, so you no longer need to toggle it manually before running a planning session.

## ✨ What's New

### Automatic Plan Mode in `/autopilot:plan`

`/autopilot:plan` now enters the harness plan mode on its own and holds for your approval before exiting. Previously, you had to manually enable plan mode before invoking the skill — if you forgot, it would run without the approval gate. Now the handoff is built in: invoke the skill, review the plan, approve, done. `/autopilot:run` is unchanged and continues to execute without a plan-approval step.

### Richer, Sharper Issue Bodies

Issues generated for GitHub and Linear now include as much context as the work actually needs — the per-section paragraph caps that previously truncated longer changes are gone. At the same time, each section (Context, What, Why, Scope, Solution) has been tightened to a single clearly-scoped job, so sections no longer repeat each other. Solution diagrams are now suggested whenever a diagram would genuinely aid understanding, not only for changes that touch multiple components.
