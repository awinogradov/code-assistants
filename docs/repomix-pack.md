# The committed Repomix pack

This repository commits a packed snapshot of `main` at `.repomix/pack.xml` and keeps it fresh with CI on every merge. Autopilot skills read this snapshot for codebase context instead of re-packing on every run. This document describes the artifact, the workflow that produces it, and the contract skills follow to consume it.

## Why it exists

Several Autopilot skills (`/plan`, `/issue-create`, `/run`, `/pr-review`, `/pr-resolve`, `/pr-answer`) need whole-codebase context to ground their output. Each previously called `mcp__repomix__pack_codebase` on every invocation, which:

- repeats the in-memory pack cost on every run, adding latency; and
- makes the skill hard-depend on the Repomix MCP server being reachable.

A committed snapshot removes the per-run pack tax, lets skills ingest a stable artifact, and produces a reviewable diff log of how the codebase shape changes over time.

## The artifact

- **Path:** `.repomix/pack.xml`
- **Format:** Repomix XML (`<file>` tags), full-repo, uncompressed.
- **Source of truth:** reflects `main` at the most recent merge commit. It is regenerated by CI, not by hand.

`.repomix/` is intentionally **not** in `.gitignore` — the pack is a committed artifact.

## How it is produced

The `.github/workflows/repomix-pack.yml` workflow regenerates the pack on every merge to `main`:

```
┌────────────┐  merged   ┌──────────────────────┐   push    ┌─────────────────┐
│  PR #N     │ ────────▶ │ repomix-pack.yml     │ ────────▶ │ main            │
│  closed    │           │ (npx repomix pack)   │           │ .repomix/       │
│ into main  │           │  detect diff         │           │   pack.xml      │
└────────────┘           └──────────────────────┘           └─────────────────┘
                                  │
                                  │ no diff
                                  ▼
                            (skip commit)
```

- **Trigger:** `pull_request` `types: [closed]`, `branches: [main]`, gated by `if: github.event.pull_request.merged == true` — runs on the merge, not on every `push`.
- **Pack:** pinned `npx repomix@<version>` (exact version, no `^`/`~`) so pack diffs reflect codebase drift, not tool-version drift.
- **Commit:** only when `git diff --quiet -- .repomix/pack.xml` reports a change; the auto-commit uses the Conventional-Commits message `chore(repomix): refresh pack from #<PR>`.
- **Push:** uses the repo's `BOT_TOKEN` secret (with `BOT_USERNAME` as the git author, matching `upstream.yml`/`release.yml`). The default `GITHUB_TOKEN` cannot push to protected `main` or trigger downstream workflows.

### Determinism

`repomix.config.json` at the repo root pins the output for a clean, reviewable diff:

- `output.filePath: ".repomix/pack.xml"` and `output.style: "xml"` — fixed location and format.
- `ignore.customPatterns: [".repomix/**"]` — excludes the pack itself, so each run does not re-pack the previously committed snapshot (which would cause recursive bloat and a perpetually-dirty diff).
- `output.git.sortByChanges: false` — Repomix defaults this to `true`, ordering files by git-change frequency, which shifts on every merge and produces noise unrelated to actual code changes.

`node_modules/`, `dist/`, `.turbo/`, and `coverage/` are excluded automatically because Repomix honors `.gitignore` and its built-in default patterns.

## How skills consume it

The 6 packing skills follow a **snapshot-first with fallback** contract:

1. Check whether `.repomix/pack.xml` exists at the repository root.
2. If it exists, call `mcp__repomix__attach_packed_output` with `path: <repo root>/.repomix/pack.xml` to obtain an `outputId`.
3. If it is absent (or the attach fails), fall back to `mcp__repomix__pack_codebase` with the skill's existing arguments.

Either path yields an `outputId`; the downstream flow (`grep_repomix_output` / `read_repomix_output`) is identical. Because the snapshot reflects `main` at the last merge, on a feature branch it lags by the in-flight changes — acceptable, since review-time skills obtain the actual diff separately and use the pack only for surrounding context.
